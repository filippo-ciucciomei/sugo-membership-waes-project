# SUGO Application Architecture

## Django Apps

rides  
membership  
notifications  

---

# Key Models

## MembershipPlan

Defines membership types.

Fields:

- name
- price
- is_active

---

## MembershipPurchase

Stores membership purchases created via Stripe webhook.

Fields:

- user
- plan
- price_paid
- created_at
- expiry_date

---

## Ride

Cycling event created by a user.

Fields:

- title
- description
- date
- location
- user (creator)

Relationships:

Ride → Attendance  
Ride → Comment  
Ride → Notification  

---

## Attendance

Join table between users and rides.

Fields:

- user
- ride
- joined_at

Relationship:

User ↔ Ride

---

## Comment

Ride comments.

Fields:

- user
- ride
- content
- created_at

---

## Notification

Stores notifications generated by user activity.

Fields:

- recipient
- actor
- ride
- type
- is_read
- created_at

Types:

ride_created  
ride_joined  
ride_left  
ride_comment  

---

# Notification System

Notifications are implemented using **Django signals**.

Signals listen for model events and automatically create notifications.

Triggers:

Ride created → notify all users  
User joins ride → notify ride creator  
User leaves ride → notify ride creator  
User comments → notify ride creator  
User comments → notify other commenters  

---

# Stripe Payment System

Uses **Stripe Embedded Checkout**.

Flow:

membership_checkout_page  
↓  
create_checkout_session  
↓  
Stripe Embedded Checkout  
↓  
Stripe webhook  
↓  
MembershipPurchase created  

Webhook endpoint:

/membership/webhook/

---

# Access Control

Users must have **active membership** to access rides.

Helper function:

user_has_active_membership(user)

Used by a MembershipRequiredMixin.

---

# Login Flow

User logs in  
↓  
PostLoginRedirectView  
↓  
If user has active membership → home  
Else → membership purchase page
